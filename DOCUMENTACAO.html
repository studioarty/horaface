PontoFace - Documentação Oficial

Bem-vindo à documentação completa do sistema PontoFace. Navegue pelas seções utilizando o menu lateral.

1. Visão Geral do Sistema

O PontoFace é um ecossistema avançado de controle de ponto e gestão de presença. Ele utiliza Reconhecimento Facial em Tempo Real (via face-api.js) para marcação biométrica precisa e sem toque. Além disso, conta com recursos inovadores de Digital Signage (Mídia Indoor) que transformam o Totem em um mural de anúncios corporativos de alto impacto visual (Cyberpunk Glassmorphism).

2. Funcionalidades Principais

  • 👤 Biometria Local: Match biométrico ultra-rápido utilizando arrays locais (Float32Array) sem a necessidade de APIs pagas, operando até mesmo offline ou em redes restritas.
  • 📺 Digital Signage 3D/RSS: Integração com feeds de notícias automáticos e modelos de transição de vídeos em background quando o Kiosk não está sendo usado.
  • 🗣️ IA de Voz (TTS): Integração com Gemini 3.1 Pro para gerar MP3 de áudio em tempo real com orientações de voz ("Ponto Registrado com Sucesso").
  • 🔄 Sincronização Hot-Polling: Sincroniza dados assincronamente (até 50MB) com alta resiliência para não gerar timeouts de rede entre Kiosk e Servidor Central.
  • 📊 Relatórios XLSX e Extratos: Geração completa em PDF e planilhas usando bibliotecas corporativas, formatados com layout contábil.
  • 🗺️ GPS Mobile: Cerca geográfica (Geofencing) para permitir pontos apenas se o funcionário estiver num raio de distância autorizado.
  • 🛡️ Segurança Anti-Escape: O software Kiosk (em Windows ou React Web) impede que funcionários saiam da tela sem a senha (PIN) de administradores.

3. Instalação: Servidor Próprio (Local/VPS)

Siga os passos abaixo para inicializar o sistema PontoFace no seu próprio servidor, seja ele Windows para rodar local na empresa, ou Ubuntu em VPS remota.

Backend (API Node.js)

cd backend
npm install
npx prisma db push  // (Sincroniza Banco SQLite)
npm run start       // (Inicia na porta 3000)

Frontend (React/Vite)

Volte para a pasta raiz, altere a variável VITE_API_URL no arquivo .env com o IP do seu Servidor Local (ex: http://192.168.0.10:3000)

npm install
npm run build      // (Gera a versão compilada estática)

Pegue os arquivos compilados da pasta raiz /dist e sirva-os com um software auxiliar, como NGINX ou um simples "serve".

4. Instalação: Hospedagem (Ex: Hostinger)

Para instalação em CPanel/hPanel na Hostinger ou HostGator (onde o acesso a terminal é limitado).

  • Criação de App Node.js: No painel da hospedagem (ex: Ferramentas -> App Node.js), aponte para a pasta do seu backend. Escolha o arquivo de inicialização server.js e clique no botão para "Instalar NPM" provido pelo próprio painel.
  • Configurando Frontend: Localmente, edite o arquivo .env e insira sua URL pública no VITE_API_URL (https://api.seudominio.com.br). Em seguida, rode npm run build.
  • Upload Estático: Pegue todo o conteúdo da pasta remanescente /dist e suba via Gerenciador de Arquivos para o seu public_html do domínio principal (https://seudominio.com.br).
  • Atenção às Rotas (htaccess): Como a aplicação é uma Single Page Application, na pasta public_html adicione o bloco abaixo no .htaccess:
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
